# DeepBVP — гибридный ИИ-решатель краевых задач

**Автор:** Черных Александр, МАИ, 3 курс
**Статус:** исследовательский проект

DeepBVP — это гибридный метод решения нелинейных краевых задач (BVP) второго
порядка с малым параметром при старшей производной. Нейросеть предсказывает
качественное начальное приближение для классического итерационного решателя
`scipy.solve_bvp`, что существенно расширяет область сходимости численного
метода — особенно в режиме сингулярного возмущения (погранслой).

Проект включает:
- обучающий пайплайн (генерация датасета, обучение, тестирование);
- модель `BVPNetTurbo` (PyTorch);
- REST API-сервис на **FastAPI**, оборачивающий модель для инференса;
- Docker-контейнер для развёртывания сервиса.

## Постановка задачи

### Математическая формулировка

Рассматривается нелинейная краевая задача второго порядка на отрезке $x \in [0, 1]$:

$$\varepsilon y'' + p(x)\ y' + q(x)\ y + j\ y^2 + k\ y^3 = f(x)$$

$$y(0) = \alpha, \qquad y(1) = \beta$$

где коэффициенты задаются параметрически:

$$p(x) = p_0 + p_1 x + p_2 x^2 + w_1 \sin(w_2 x) + v_1 \cos(v_2 x)$$

$$q(x) = q_0 + q_1 x + q_2 x^2 + e_1 \sin(e_2 x) + u_1 \cos(u_2 x)$$

$$f(x) = A \exp\\left(-\left(\frac{x - \mu}{\sigma}\right)^{2}\right) + c_0 + c_1 x + c_2 x^2$$

Итого задача описывается **25 входными параметрами**.

### В чём сложность

`ε ∈ [10⁻⁴, 10⁻¹]` — малый параметр при старшей производной. При $\varepsilon \to 0$
возникает **сингулярное возмущение**: решение образует погранслой — узкую
область резкого изменения функции шириной порядка $\varepsilon$ у одной из границ.

Классический решатель `scipy.solve_bvp` требует начального приближения.
Стандартная линейная интерполяция между граничными значениями:

$$y_{\text{zero}}(x) = \alpha + (\beta - \alpha)\ x$$

не содержит структуры погранслоя, и при малых $\varepsilon$ итерационный процесс
расходится — решатель возвращает ошибку.

### Цель проекта

Обучить нейросеть, которая по 25 параметрам задачи предсказывает
**структурно верное** начальное приближение — такое, которое уже содержит
погранслой и выводит решатель в область сходимости.

---

## Идея решения

### Гибридный подход

Вместо обучения нейросети решать уравнение напрямую (как в PINNs), сеть
используется **только для генерации информированного старта**, который затем
уточняется классическим `scipy.solve_bvp`.

Преимущества такого подхода:

- сохраняются все гарантии численного метода (оценка погрешности, устойчивость);
- сети не нужна высокая точность — достаточно уловить структуру (погранслой);
- обучение идёт на метках того же численного метода — ошибки не накапливаются;
- модель обобщается на более жёсткие режимы за счёт структурных закономерностей.

Алгоритм работы:

1. Вектор параметров $\theta = (\varepsilon, p_0, \ldots, c_2) \in \mathbb{R}^{25}$ подаётся на вход;
2. Нейросеть $F(\theta)$ предсказывает 16 коэффициентов $c = (c_1, \ldots, c_{16})$;
3. Строится начальное приближение $y_0(x) = \sum_{i=1}^{16} c_i B_i(x)$ через кубические
   B-сплайны на равномерной сетке;
4. `scipy.solve_bvp` получает $y_0(x)$ и аналитическую $y_0'(x)$ и итерирует
   до заданной точности.

### Почему B-сплайны

- **Компактность**: 16 коэффициентов достаточно, чтобы закодировать гладкую
  функцию на `[0, 1]` с одним резким переходом;
- **Гладкость**: непрерывные производные до 2-го порядка — нужны для невязки
  и передачи `y'` в солвер;
- **Линейность по коэффициентам**: упрощает задачу до обычной регрессии
  16 чисел вместо предсказания значений на сетке из 500 точек.

## Архитектура нейросети (BVPNetTurbo)

Полносвязная сеть с residual-соединениями, **406 032** обучаемых параметра.

```
Вход: 25 признаков
  ↓
Linear(25 → 256) → GELU → BatchNorm
  ↓
ResidualBlock(256) × 3
  (каждый блок: Linear → GELU → Linear, + skip-connection, затем GELU)
  ↓
Linear(256 → 16)
  ↓
Выход: 16 коэффициентов B-сплайна
```

**Ключевые решения (с честной оценкой обоснованности):**

| Решение | Обоснование |
|---|---|
| GELU вместо ReLU | Общее наблюдение из литературы; прямого A/B-сравнения на этом датасете не проводилось |
| Residual-соединения | Классический приём против затухания градиентов; изолированного теста «с/без» не было |
| BatchNorm только на входе | Лучший результат по итогам подбора гиперпараметров (раздел hyperparameter search) |
| Без Dropout | При dropout 0.1+ R² не улучшался или снижался — эмпирический вывод для данного датасета |

### Обучение

- Нормализация: `StandardScaler` для входа, `RobustScaler` для целевых
  коэффициентов (устойчив к выбросам в метках);
- Функция потерь: MSE между предсказанными и истинными коэффициентами;
- Оптимизатор: `AdamW`, `lr=1e-3`, `CosineAnnealingLR` на 500 эпох, батч 128;
- Ранняя остановка: 100 эпох без улучшения на валидации.

### Подбор гиперпараметров

Перебирались: глубина (1–5 блоков), ширина (128/256/512), активация
(ReLU/GELU/Swish), наличие BatchNorm, learning rate (1e-4…3e-3), Dropout
(0.0/0.1/0.2). Лучшая конфигурация: **3 блока × 256 нейронов, GELU, BatchNorm
только на входе, без Dropout, lr = 1e-3**.

## Датасет

Параметры сэмплировались случайным равномерным сэмплированием из физически
обоснованных диапазонов. Каждая задача решалась в два этапа (`tol=1e-3` →
грубое решение → `tol=1e-5` → точное решение) для получения точных меток.

| Показатель | Значение |
|---|---|
| Попыток генерации | 50 000 |
| Успешно решено | 36 141 (72.3%) |
| После фильтрации выбросов (`\|c\| > 10`) | 35 952 |
| Обучающая выборка (85%) | 30 559 |
| Тестовая выборка (15%) | 5 393 |
| Сетка решения | 500 узлов |
| Точность меток | `tol = 1e-5` |

Диапазоны параметров: `ε ∈ [1e-4, 5e-2]` (лог-шкала), `α, β ∈ [-2, 2]`,
`p0..p2, q0..q2 ∈ [-3, 3]`, `j, k ∈ [-0.8, 0.8]`, `A ∈ [-4, 4]`.

### Расширение области сходимости

Два независимых теста по 5000 задач (`tol=1e-5`, `seed=42`):

**Тест 1 — in-distribution:**

| Показатель | Zero (линейный) | DeepBVP |
|---|---|---|
| Success Rate (все задачи) | 63.8% | 74.9% |
| Success Rate (решаемые задачи) | 84.6% | 99.3% |
| Спасено / потеряно задач | — | 581 / 25 |
| Соотношение побед/поражений | — | 23.2 : 1 |

**Тест 2 — out-of-distribution** (`ε` форсировано в `[5e-5, 8e-4]`,
на порядок жёстче обучающего диапазона):

| Показатель | Zero (линейный) | DeepBVP |
|---|---|---|
| Success Rate (все задачи) | 29.9% | 39.4% |
| Success Rate (решаемые задачи) | 74.1% | 97.8% |
| Спасено / потеряно задач | — | 521 / 44 |
| Соотношение побед/поражений | — | 11.8 : 1 |

**Вывод:** DeepBVP не ускоряет решение там, где линейный старт и так
сходится (среднее число итераций одинаково). Ценность — именно в расширении
области сходимости: сеть выучила структурные закономерности (положение и
крутизну погранслоя), а не запомнила примеры, что подтверждается устойчивой
работой при `ε`, отсутствовавших в обучающей выборке.

## Сравнение с альтернативными подходами

**Классический ML** (Random Forest, XGBoost, CatBoost) — `R² < 0.6`: деревья
плохо аппроксимируют непрерывные нелинейные зависимости между параметрами
задачи и весами базиса.

**PINNs** (Physics-Informed Neural Networks) — решают уравнение напрямую, без
солвера. Ограничения: плохо работают при малых `ε` без спецтехник, требуют
переобучения/оптимизации под каждую новую задачу, не дают гарантий точности.
DeepBVP сохраняет все гарантии классического численного метода, лишь улучшая
старт.

## Ограничения и планы развития

**Текущие ограничения:**
- модель обучена только на конкретном классе уравнений (форма (1) выше);
- при `ε < 10⁻⁴` часто не сходится ни один из методов — предел решателя, а не модели;
- 16 B-сплайнов дают `R² ≈ 0.86`; для более резких погранслоёв может
  потребоваться адаптивный базис.

**Направления развития:**
- адаптивный базис (wavelets / адаптивные узлы) для сгущения в области погранслоя;
- физически-информированная дообучка с добавлением невязки уравнения в лосс;
- расширение класса уравнений с маршрутизацией через LLM-классификатор типа уравнения.

## Установка и запуск

### Локально (без Docker)

```bash
python -m venv venv
source venv/bin/activate        # Windows: venv\Scripts\activate
pip install -r requirements.txt

# обучающий пайплайн
python generate_dataset_V4.py
```

`joblib`, `streamlit`, `fastapi`, `uvicorn`.

## API-сервис (FastAPI)

Модель `BVPNetTurbo` обёрнута в REST API на FastAPI. Сервис принимает 25
параметров краевой задачи и возвращает предсказанные коэффициенты
B-сплайнов и/или готовое начальное приближение для `solve_bvp`.

### Эндпоинты

| Метод | Путь | Описание |
|---|---|---|
| `GET` | `/health` | проверка работоспособности сервиса |
| `POST` | `/predict` | предсказание 16 коэффициентов B-сплайна по 25 параметрам задачи |
| `GET` | `/docs` | автогенерируемая Swagger-документация FastAPI |
